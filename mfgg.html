In object oriented programming, each object will have external interfaces through which it can be made use of.

There is no need to look into its inner

details.

The object itself may be made of many smaller objects again with proper interfaces.

The user needs to know the external interfaces only to make use of an object.

The internal details of the objects are hidden which makes them abstract.

The technique of hiding internal details in an object is called data abstraction.
